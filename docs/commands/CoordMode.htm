CoordMode

다양한 명령어에 대하여 화면 또는 활성 창에 좌표 모드가 상대적이 되도록 설정합니다.

CoordMode, ToolTip|Pixel|Mouse|Caret|Menu [, Screen|Window|Client]

매개변수

Param1

ToolTip: ToolTip에 영향을 줍니다.

Pixel: PixelGetColor, PixelSearch, 그리고 ImageSearch에 영향을 줍니다.

Mouse: Affects MouseGetPos, Click, 그리고 MouseMove/Click/Drag에 영향을 줍니다.

Caret: 내장 변수 A_CaretX와 A_CaretY에 영향을 줍니다.

Menu: 좌표가 지정되어 있지 않을 때 Menu Show에 영향을 줍니다.

Param2

Param2가 생략되면, Screen이 기본값입니다.

Screen: 좌표는 바탕화면에 상대적입니다 (전체 화면).

Relative: 좌표는 활성 창에 상대적입니다.

Window [v1.1.05+]: Relative와 동등하며 명료하기 때문에 이것을 권장합니다.

Client [v1.1.05+]: 좌표는 활성 창의 클라이언트 영역에 상대적입니다. 클라이언트 영역에서 제목 바, 메뉴 (표준 메뉴라면) 그리고 테두리는 제외됩니다. 클라이언트 좌표는 OS 버전와 테마에 의존하지 않습니다.

논평

이 명령어가 사용되지 않으면, 모든 명령어는 따로 문서화되어 있는 명령어를 제외하고 (예, WinMove 그리고 InputBox) 활성 창에 상대적인 좌표를 사용합니다.

새로 기동된 쓰레드는 (핫키, 맞춤 메뉴 항목, 또는 타이머 서브루틴) 이 명령어에 대한 기본 값을 가지고 새롭게 시작합니다. 그 기본 값은 자동-실행 섹션 (스크립트 상단)에 이 명령어를 사용하여 바꿀 수 있습니다).

관련 항목

Click, MouseMove, MouseClick, MouseClickDrag, MouseGetPos, PixelGetColor, PixelSearch, ToolTip, Menu

예제

CoordMode, ToolTip, Screen  ; 툴팁을 절재 화면 좌표에 배치합니다:
CoordMode, ToolTip  ; 위와 같습니다. 똑 같이 "screen"이 기본값입니다.
